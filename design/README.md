# Design Workflow

This repo stores design research, approval artifacts, and handoff documents under `design/`.

## Naming

Use `YYYY-MM-DD-short-topic.md` for new artifacts so files sort chronologically.

## Directories

- `research/`: discovery reports, screenshots, and clustered findings
- `patterns/`: reusable layout, onboarding, dashboard, and motion recommendations
- `boards/`: decision boards, look-lock summaries, and concept board notes
- `specs/`: approved UX specs and prompt-ready structure documents
- `tokens/`: locked design systems and token definitions
- `prototypes/`: Stitch prompts, prototype reviews, and concept decisions
- `wireframes/`: low-fidelity text or ASCII wireframes
- `figma/`: Figma links, review notes, approvals, and analyzer inputs
- `analysis/`: Figma MCP analysis reports and design drift reviews
- `interactions/`: CodePen interaction references and implementation caveats
- `handoff/`: implementation briefs and review notes
- `style-guide/`: reusable style guides and long-term design reference docs
- `templates/`: starter documents for each phase

## Default sequence

1. Research patterns and save the report in `research/`.
2. Build and approve a decision board in `boards/`.
3. Draft and approve the UX spec in `specs/`.
4. Lock tokens and motion rules in `tokens/`.
5. Save look-lock and concept review artifacts in `boards/` or `prototypes/`.
6. Save Stitch prompts and prototype decisions in `prototypes/`.
7. Draft wireframes in `wireframes/`.
8. Record Figma review links and notes in `figma/`.
9. Save Figma analysis and drift checks in `analysis/`.
10. Save interaction references in `interactions/`.
11. Save implementation briefs and review notes in `handoff/`.
12. Save reusable style guides in `style-guide/`.

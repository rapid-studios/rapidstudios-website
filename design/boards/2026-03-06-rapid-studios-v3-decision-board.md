## Decision Sections

- Decision area: Hero style
- Option references:
  - editorial hero with proof rail
  - compact conversion hero
  - balanced service-led hero
- Recommended option:
  - oversized editorial hero with short support copy, dual CTA row, and proof-oriented right rail
- Reasoning:
  - This is still the strongest overlap between the approved research, the existing homepage blueprint, and the intended premium studio tone.
- Final choice:
  - Approved: Option A, oversized editorial hero with short support copy, dual CTA row, and proof-oriented right rail

- Decision area: Navigation identity
- Option references:
  - pill-shell sticky header with dual CTA cluster
  - flatter utility header
  - ultra-minimal short nav
- Recommended option:
  - pill-shell sticky header with `Work`, `Services`, `Process`, `Contact`, plus visible CTA actions
- Reasoning:
  - The shell gives the site stronger identity and supports the short, conversion-aware navigation model already validated in research.
- Final choice:
  - Approved: Option A, pill-shell sticky header with `Work`, `Services`, `Process`, `Contact`, plus visible CTA actions

- Decision area: Trust and card simplicity
- Option references:
  - compact trust strip plus simple supporting cards
  - heavier proof block
  - more visual-first proof modules
- Recommended option:
  - compact trust strip near the top with quiet supporting cards and later testimonials
- Reasoning:
  - This preserves the selective homepage rhythm while still building credibility early and often.
- Final choice:
  - Approved: Option A, compact trust strip near the top with quiet supporting cards and later testimonials

- Decision area: Featured work treatment
- Option references:
  - editorial case-study cards
  - denser project grid
  - single oversized showcase
  - tag-heavy proof cards
- Recommended option:
  - editorial cards with one media well, concise title, and one-line outcome
- Reasoning:
  - Featured work should read like curated proof, not a portfolio index or dashboard.
- Final choice:
  - Approved: Option A, editorial cards with one media well, concise title, and one-line outcome

- Decision area: Homepage process density
- Option references:
  - 3-step homepage summary
  - denser process explanation
  - vertical timeline treatment
  - fuller story on the dedicated `/process` page
- Recommended option:
  - 3-card homepage process with the fuller narrative reserved for the process page
- Reasoning:
  - The homepage should stay fast and confidence-building. Deeper explanation belongs one layer down.
- Final choice:
  - Approved: Option A, 3-card homepage process with the fuller narrative reserved for the process page

- Decision area: Offer framing and CTA path
- Option references:
  - engagement framing plus email-first CTA
  - project-start-only CTA
  - trust-led closing pattern
  - form-first closing path
- Recommended option:
  - premium engagement framing with an email-forward closing CTA and secondary work path
- Reasoning:
  - This gives visitors enough buying context without making the site feel commoditized or forcing a heavy form too early.
- Final choice:
  - Approved: Option A, premium engagement framing with an email-forward closing CTA and secondary work path

- Decision area: Typography, color, and state priority
- Option references:
  - `Space Grotesk` + `Manrope`
  - neutral fallback styling
  - explicit loading, empty, success, and error states
  - defer state design
- Recommended option:
  - `Space Grotesk` for display and headings, `Manrope` for body and UI copy, restrained blue-led light mode, and required state coverage in the next gate
- Reasoning:
  - These are the highest-leverage system choices for reducing drift between research, Figma, and implementation.
- Final choice:
  - Approved: Recommended direction, `Space Grotesk` for display and headings, `Manrope` for body and UI copy, restrained blue-led light mode, and required state coverage in the next gate

- Decision area: Motion style
- Option references:
  - section reveal
  - hover lift
  - header condense
  - higher-energy motion language
- Recommended option:
  - subtle reveal, hover lift, and header condense with reduced-motion fallbacks
- Reasoning:
  - The motion system should reinforce hierarchy and polish, not become the main event.
- Final choice:
  - Approved: Recommended motion direction, subtle reveal, hover lift, and header condense with reduced-motion fallbacks
